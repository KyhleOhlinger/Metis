Welcome to Metis

Metis is a local-first, AI-augmented personal knowledge base built for writers, engineers, and thinkers. Your notes live as plain .md files on your own filesystem — no cloud sync, no lock-in, no subscription.

💡 All your data stays on your machine. AI calls only send the specific note content needed for each task — your full vault is never transmitted in a single request.
Three-pane layout
Sidebar file tree · CodeMirror editor · Command Center — all resizable and collapsible.
Source · Visual + Planner access
Edit raw markdown, switch to HTML preview, and open Planner from the button at the bottom of the left sidebar.
AI writing assistant
Persona-driven AI that can read, write, and update your notes — with a review step before any changes land on disk.
Plain-text files
Everything is .md on disk. Open your vault in any editor, sync it with Git, or back it up with Time Machine.

First launch

  1. Click File → Open Vault… (or ⌘O) and pick a folder to use as your vault.
  2. Metis creates daily/, meetings/, summaries/, and assets/ sub-folders automatically.
  3. Create your first note with ⌘N or the + button in the sidebar.
  4. To use AI features, add at least one API key in Command Center → Settings → API Providers.

Vaults

A vault is any folder on your filesystem. Metis remembers your last opened vault and restores it on every launch.

Create a new vault

Go to File → New Vault…. Pick a parent folder, name your vault, and Metis sets up the default folder structure for you.

Open an existing vault

Use File → Open Vault… (⌘O) and pick any folder. Metis walks the directory tree and shows all .md files in the sidebar.

Multiple vaults

Opening a different vault while one is already loaded spawns a new Metis window, keeping both vaults accessible simultaneously. Each window is fully independent.

Writing & Editing

Metis uses CodeMirror 6 with full GitHub-Flavored Markdown support. All standard markdown syntax works — headings, bold, italic, tables, task lists, fenced code blocks, and more.

Source mode vs Visual mode

Switch using the toggle in the editor header bar, or via View → Source Mode / Visual Mode.

  • Source mode — raw markdown with syntax highlighting, inline image rendering, and Cmd+Click link following.
  • Visual mode — rendered HTML preview (tables, callouts, task checkboxes, wikilinks are all interactive).

Formatting toolbar

The toolbar above the editor (visible in Source mode) provides quick access to:

H1 · H2 · H3 · Bold · Italic · Inline code · Link · Image · Code block · Blockquote · Bullet list · Numbered list · Horizontal rule

Slash menu

Type / at the start of a blank line to open the block-insertion menu. Choose headings, lists, code blocks, callouts, or a horizontal rule without leaving the keyboard.

Callout blocks

Obsidian-style callouts are supported:

  • > [!INFO] — blue information callout
  • > [!TIP] — green tip callout
  • > [!WARNING] — yellow warning callout
  • > [!DANGER] — red danger callout

Smart paste

  • URL over selected text — automatically wraps into a Markdown link [selected text](url).
  • Screenshot / image from clipboard — saves to assets/ and inserts ![](assets/filename.png).

Auto-save

Notes are saved automatically 1 second after your last keystroke. The save status is shown in the editor header bar. You can also save manually with ⌘S.

Background themes

Click the colour swatch in the editor header to switch between Dark, Black, Slate, White, and Cream — applied instantly without reloading the editor.

Planner

Open Planner using the Planner button at the bottom of the left files sidebar. It provides a weekly work grid, weekly/monthly reflections, reusable planning templates, and a PTO & events tracker with automatic Daily Log sync.

Planner tabs

  • Daily Log — Monday-Friday planning and work-log grid.
  • Weekly Review — week-by-week retrospective notes with action points.
  • Monthly Review — reflection prompts and month-level review notes.
  • Templates — recurring planning templates (daily/weekly/monthly/interval) with recurrence day controls.
  • PTO & Events — Public Holidays, PTO, Conferences, and Office Trips tracker tables.

Template behavior

  • Templates auto-fill empty "What do I want to do" cells only.
  • Disabling or deleting a template asks for a Final Day and removes future template-generated occurrences after that date.
  • Template-generated content is tracked so manual edits are not unintentionally removed.

PTO & Event tracker sync

  • Public Holidays (single date) set matching Daily Log cells to Public Holiday.
  • PTO (start/end range) sets matching weekday cells to PTO.
  • Conferences (start/end range) set matching weekday cells to Off-site/Conference.
  • Office Trips (start/end range) add a non-destructive banner to the top of each matching day cell (does not replace the block).
  • Tracker-controlled cells show an Edit Event link to jump back to the matching tracker row.

PTO counter

The tracker includes a running PTO summary: PTO Remaining = Total Allocation − sum(Days Taken).

Search & Replace

Metis provides two levels of search: in-editor (single file) and vault-wide (all notes).

In-editor find & replace

Press ⌘F to open the find panel at the top of the editor. Press ⌘H to open find-and-replace.

ActionShortcut
Find in file⌘F
Find and replace⌘H
Next matchF3 or Return
Previous match⇧F3
Close search panelEsc

The in-editor search panel supports case-sensitive and regex toggles, plus Replace and Replace All buttons.

Vault-wide search & replace

Press ⌘⇧F or click the search icon in the sidebar header to open the vault-wide search panel. This replaces the file tree with a full-text search view.

  • Type a query — results update automatically after a short debounce (300 ms). Searches both file names and file contents.
  • Toggle case-sensitive (Aa) and regex (.*) modes using the icons next to the search input.
  • Results are grouped by file, each showing line numbers and matching text with highlights. Files whose names match the query are highlighted with an accent border.
  • Click any content result to open that file in the editor and jump to the matching line. Click a filename-only match to open the file.
  • Click the replace icon to expand the Replace input, then use Replace All to perform a vault-wide find-and-replace across all notes.
⚠️ Vault-wide Replace All writes changes directly to disk for every affected file. There is no undo across multiple files — consider using version control (Git) as a safety net.

To return to the file tree, click the back arrow at the top of the search panel, press Esc, or click the search icon again.

Metadata & Properties

Every note can carry structured metadata stored as YAML frontmatter at the top of the file. The Metadata Panel below the editor lets you edit it without touching raw YAML.

Built-in fields

FieldDescription
Statusdraft · in-progress · review · done · archived — shown as a coloured icon in the sidebar
DateISO date stored as date: in frontmatter; editable via calendar picker
ParentAuto-derived from the containing folder — read-only
AliasesAlternative names for the note; searched by [[wikilink]] autocomplete and the Command Palette
TagsEditable tag chips backed by tags: frontmatter
LinksRead-only list of all [[wikilinks]] found in the note body
ℹ️ Status colours update instantly as you type — you don't need to save the file for the sidebar icon to reflect your current frontmatter value.

Navigation

Quick Switcher

Press ⌘P to open the fuzzy note switcher. It searches note names and aliases, and can be filtered by status using the chips at the top.

Wikilinks

Type [[ in Source mode to open a live autocomplete dropdown of all notes in your vault (searching both names and aliases). In Visual mode, wikilinks are clickable and navigate to the linked note.

Daily note

Click the calendar icon in the sidebar header (or press ⌘D) to create or open today's daily note at daily/YYYY-MM-DD.md.

Sidebar

  • Collapse or expand with ⌘\ or View → Toggle Sidebar.
  • Right-click any file or folder for a context menu: rename, delete, run with a persona, reveal in Finder, copy path.
  • Drag a file onto a folder to move it. Drag a file onto a persona chip in the sidebar to scope an instant AI run.

Command Center — AI Tab

The AI tab in the right Command Center panel is where you interact with your AI personas. Open it with ⌘⇧\ or the panel toggle button.

Running a prompt

  1. Select a persona chip at the top of the AI tab.
  2. Choose a scope — Current File, a Specific File, a Folder, or the Full Vault.
  3. Type your prompt and press Return or click Run.
  4. A spinning indicator with "{Persona} is thinking…" appears while the model processes your request. The response then streams in real-time. If the AI calls an agent file tool, a review step appears before any changes are written to disk.

Agent file tools

When running against a folder or full vault, the AI can:

  • Write — fully replace the current note
  • Append / Prepend — add content to the end or beginning
  • Insert at cursor — insert at your current cursor position
  • Create new note — create a new .md file anywhere in the vault

System Default personas

  • The Librarian — scans your vault for orphaned notes (no incoming or outgoing links) and suggests connections.
  • Task Manager — aggregates all - [ ] checkboxes across your vault into summaries/todo.md.

Context strategy

Metis automatically picks the most relevant notes to send to the AI without exceeding the model's context limit:

  • Direct — all notes fit within budget; everything is sent as-is.
  • TF-IDF — notes are scored against your prompt; only the highest-scoring ones are sent.
  • Scout — a fast preliminary AI call pre-filters candidates from your vault; only selected files are retrieved in full.

Personas

A persona is a named AI configuration with its own system prompt, model, and provider. Create as many as you need for different writing tasks.

Creating a persona

  1. Go to Command Center → Settings → Personas.
  2. Click + New Persona.
  3. Fill in a name, icon (emoji), system prompt, and select a provider.
  4. Click ↓ Models to fetch the latest available models for that provider, or type a model name directly.
  5. Save the persona — it appears immediately in the AI tab chip bar.

Enabling / disabling personas

Toggle the switch next to any persona to hide or show it in the chip bar. Disabled personas are preserved but not visible during AI sessions.

Linking a persona to a Quick Action

When editing a quick action (see below), use the persona picker to route that action to a specific persona. If none is selected, the quick action uses whichever persona is currently active in the AI tab.

Quick Actions

Quick Actions appear in the floating toolbar that pops up whenever you select (highlight) text in Source mode. They let you instantly apply AI transformations to selected text.

Built-in quick actions

ActionWhat it does
ImproveRewrites the selection for clarity and flow
SummariseCondenses the selection into a shorter form
ExplainExplains the selected text in plain language
ExpandElaborates on the selection with more detail
ActionsExtracts action items from the selected text

Managing quick actions

Go to Command Center → Settings → Quick Actions to:

  • Add a new action with a custom label and prompt.
  • Edit the label, prompt, or assigned persona of any action.
  • Reorder actions by dragging the grip handle on the left.
  • Delete an action you no longer need.
💡 Assign a dedicated persona to a quick action to ensure it always uses a specific model and system prompt, regardless of what's selected in the AI tab.

AI Providers & Settings

All AI providers use an OpenAI-compatible API. Add your own API key for any provider in Command Center → Settings → API Providers.

ProviderKey formatHighlights
OpenAI sk-… GPT-4o, o1, o3 series; best general-purpose reasoning
Google Gemini AIzaSy… Gemini 2.0 Flash, 1.5 Pro — up to 1 M token context window
Groq gsk_… Llama 3.x, Mixtral — extremely fast inference
Perplexity AI pplx-… Sonar family with real-time web search integration
⚠️ API keys are stored in your OS app-data directory (~/Library/Application Support/com.metis.app/ on macOS) and are never sent anywhere except to your chosen AI provider.

Testing a connection

After entering an API key, click Test Connection. Metis makes a zero-cost models.list() call to verify the key is valid and the provider is reachable. Detailed error messages are shown if it fails.

Fetching models

When creating or editing a persona, click ↓ Models to pull the latest available models directly from the provider. Metis displays up to 5 curated models and caches the result. You can also type any model name manually in the input box.
